Flexibility

What Does it Mean for Software to be Flexible?

  • Portability (supports multiple platforms)
    •
  • Evolves well over time
    •
    • programming vs. software engineering
  • Easy to change (short turnaround time)
    •
  • Behaviour is adaptable and dynamic
    •
  • Copes with future unknowns
    •

Example

Blu Wireless

  • IP networking over 5G mmWave (60 GHz) modems
    •
    • 802.11ad MAC + PHY (Hydra)
  • Embedded ARM based Cavium (Marvell) NPUs
    •
    • High specification running Ubuntu 18.04
  • Multi gigabit wireless links (up-to 1km)
    •
    • Mobile and Fixed Wireless Applications

High Speed Rail

  • High-bandwidth, low latency Internet connectivity
    •
    • Connection must not drop e.g. in tunnels
  • mmWave radios are mounted along the track-side
    •
    • 10Gb fibers provide back-haul to the Internet
  • mmWave radios are also mounted on the trains
    •
    • NPUs co-ordinate via on-board wired Ethernet

Connection Manager

  • Critical software component
    •
  • Decides which radio to connect to and when
    •
  • v1.0 implemented simple behaviour:
    •
    1. Scan for track-side access points (APs)
    2. Connect to AP with the strongest signal
    3. Wait for disconnection (and go back to 1)

Observations

  • Fixed behaviour was too simplistic
    •
    • RF anomalies led to poor decision making
  • Code could only be extended by Software Engineers
    •
    • Modifications required C++ expertise
  • Problems were 'worked around' and not 'solved'
    •
    • New requirements could not be accomodated

Improvements could not be made fast enough

Mobile Connection Manager

  • Complete redesign of Connection Manager
    •
  • Decoupled modular architecture (C++17 & Lua 5.3)
    •
  • Actions (C++17): 'Scan', 'Connect', 'Probe' etc.
    •
  • Behaviours (Lua): sequences of actions
    •
    • Maximise bandwidth
    • Implement 'beam choreography'

How to Combine C++17 and Lua 5.3

LuaChat

  • Unix 'talk' program (written in C++ & Lua)
    •
  • Available on GitHub (MIT license)
    •
  • Asio for asynchronous TCP/UDP communications, timers and POSIX signal handling
    •
  • spdlog for logging, cxxopts for command line processing and CMake for build generation
    •

Build Instructions

Ubuntu 18.04 (Linux Mint 19): 


git clone https://github.com/jamespascoe/LuaChat.git
sudo apt-get install lua53 luarocks swig
luarocks install luaposix
mkdir build; cd build; cmake ../LuaChat; make

MacOS (Catalina): 


git clone https://github.com/jamespascoe/LuaChat.git
brew install lua luarocks swig
luarocks install luaposix
mkdir build; cd build; cmake ../LuaChat; make
